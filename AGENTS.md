# AGENTS.md

## Что это

Экосистема строительных калькуляторов «Мастерок»: точные расчёты материалов для реального ремонта.

- **Этот репозиторий** — web: Next.js 15 (App Router, SSR), Tailwind 4, TypeScript. Прод: https://getmasterok.ru на Timeweb Cloud Apps (`next start`).
- **Мобильное приложение** — Flutter, отдельный репозиторий `C:\probrab1`, со своим AGENTS/CLAUDE.md.
- Язык интерфейса и контента — русский. Рынок — Россия.

Ядро ценности: ~70 калькуляторов, которые считают корректно, одинаково на web и mobile, и дают «итог к покупке», а не абстрактную цифру.

## Приоритеты (по убыванию)

1. Корректность расчётов
2. Паритет логики web ↔ mobile
3. Практическая применимость результата (реальные мешки/листы/рулоны)
4. Простота ввода → 5. Понятность результата → 6. Чистота архитектуры → 7. Производительность → 8. Визуальная аккуратность

Конфликт «красивый код vs правильный расчёт» решается в пользу расчёта.

## Расчёты — железные правила

- **Не выдумывать** формулы, коэффициенты, нормы (ГОСТ/СНиП/СП), типовые расходы. Нет данных — явно пометить допущение.
- Этапы расчёта разделены и не смешиваются в одну цифру: нормализация ввода → единицы → чистая математика → строительные поправки (подрезка, неровности, слои) → запас → округление → упаковка → «итог к покупке».
- Точная потребность и «к покупке» — два разных значения, оба показываем.
- Штучные единицы (мешки, рулоны, листы, шт) — всегда целые, округление вверх.
- Округлять только на последнем этапе; внутри считать точно.
- Коэффициенты и упаковки — в конфигах/справочниках, не магическими числами в коде.
- **Изменение формулы/запаса/округления/упаковки/дефолтов = обязательные тесты + явная фиксация в отчёте** (старая логика → новая → что изменится для пользователя). Принесённый пользователем баг сначала фиксируется тестом.
- Расчётная логика не живёт в React-компонентах — только в `engine/` и `src/lib/`.

## Паритет web ↔ mobile

Если задача касается формул, полей, дефолтов, упаковок или структуры результата:

1. Источник истины — `engine/` canonical-спеки.
2. После правки: `npm run sync:all` (перегенерит Dart-спеки, parity-фикстуры, meta).
3. `npm run test:parity` должен быть зелёным.
4. Расхождение с mobile — это баг: зафиксировать, определить источник истины, унифицировать.

## UX и язык

- Всё пользовательское — на русском, без канцелярита и AI-штампов.
- Минимум обязательных полей, разумные дефолты, подсказки у неоднозначных полей.
- Ошибки объясняют: что не так, почему, как исправить.
- При правках UI проверять: пустое состояние, длинные значения, мобильную ширину, переполнения, тёмную тему.
- Результат объясним: что считали, какой запас, как округлили, сколько покупать.

## SEO / контент

- Тексты для людей; без переспама ключей; страница не обещает того, чего не делает.
- Meta калькуляторов генерируются (`sync:meta`), руками `meta.generated.ts` не трогать.
- На сайте есть `llms.txt`, RSS, sitemap, JSON-LD (`src/components/seo/`) — при добавлении страниц не забывать про них.

## Деплой и известные грабли

- Деплой: Timeweb Cloud Apps, сборка из git. **Untracked-файл = файла нет на проде** (был реальный инцидент с 404 агента). Перед «выкатили» проверять `git status`.
- Env-переменные агента (DEEPSEEK_API_KEY и опциональные) — `docs/mikhalych-timeweb-deploy.md`.
- Анти-DDoS Timeweb режет иностранные дата-центровые IP: таймауты из-под VPN/зарубежных проверялок ≠ «сайт упал». Проверять доступность через PageSpeed/GSC/RU-ноды.
- `CLAUDE.md` — это импорт этого файла (`@AGENTS.md`). Правки промпта делать здесь, один раз.
