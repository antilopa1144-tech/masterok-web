# Calc Core Skill

## Назначение
Этот навык отвечает за проектирование, аудит, исправление и унификацию строительных расчётов.

## Главная цель
Сделать так, чтобы:
- математическая логика была корректной
- результат был реалистичным для стройки
- поведение совпадало на web и mobile
- расчёты были объяснимыми
- пользователь получал полезный итог

## Обязательная методика работы
1. Найди текущую формулу и все места её использования.
2. Определи источник истины.
3. Выпиши входные параметры, единицы, промежуточные вычисления, правила запаса, округления и упаковки.
4. Проверь, не смешаны ли чистая математика, строительная поправка, запас, округление и кратность покупки.
5. Если смешаны — раздели их.
6. Только потом вноси изменения.
7. После правки обязательно проверь тестами и ручной логикой, что результат стал корректнее.

## Стандарт расчётного пайплайна
1. Нормализация входных данных
2. Проверка и преобразование единиц измерения
3. Чистый математический расчёт
4. Практические строительные поправки
5. Применение запаса
6. Округление до покупаемой единицы
7. Формирование понятного результата

## Что категорически запрещено
- Выдумывать формулы и коэффициенты
- Округлять в середине вычислений без необходимости
- Смешивать запас с базовым значением в одну цифру
- Возвращать пользователю только итог без пояснения
- Использовать разную логику на сайте и в приложении
- Подменять инженерный расчёт приблизительной оценкой на глаз
- Прятать важную расчётную логику в UI

## Правила единиц измерения
Всегда явно проверяй и фиксируй смысл единиц: мм, см, м, м², м³, л, кг, шт, листы, рулоны, мешки, канистры, упаковки.

## Правила запаса
Разделяй:
- базовую потребность
- запас
- рекомендуемое количество
- итог к покупке

## Правила упаковок и покупки
Расчёт должен идти так:
1. точная потребность
2. запас
3. кратность упаковки
4. итог к покупке

## Проверка web и mobile
Если задача касается синхронизации, сравни поля, типы данных, дефолты, коэффициенты, округления, формат результата, упаковки и запас.

## Минимальный набор проверок после правки
1. типовой сценарий
2. маленькие значения
3. большие значения
4. дробные значения
5. нулевые или пограничные случаи
6. сценарий с упаковкой
7. сценарий с запасом
8. регрессию на найденный баг
