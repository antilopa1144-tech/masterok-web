# Next.js Web Skill

## Назначение
Этот навык нужен для задач по web-части проекта на Next.js.

## Абсолютные правила
- Не держи доменную логику в UI
- Не дублируй общую логику
- Русский язык обязателен
- Адаптивность обязательна
- Результат должен быть читаемым

## Как действовать при web-задачах
1. Найди экран, страницу, компонент или модуль.
2. Определи, где UI, а где логика.
3. Проверь, не смешана ли логика расчёта с отображением.
4. Если смешана — по возможности раздели.
5. Внеси минимально опасные изменения.
6. Проверь адаптивность и отображение результата.
7. Проверь, не пострадала ли интеграция с общей логикой.

## Что обязательно проверять в UI
- пустое состояние
- частично заполненную форму
- ошибки валидации
- очень большие числа
- дробные значения
- длинные подписи
- длинные результаты
- мобильную ширину
- отсутствие переполнения
- отсутствие обрезания текста
- отсутствие скачков layout
