# Project Workflow Skill

## Назначение
Этот навык нужен, чтобы агент не работал хаотично и не переписывал проект кусками без понимания общей картины.

## Базовый порядок работы
1. Найди все затронутые файлы.
2. Определи источник истины.
3. Пойми границы задачи.
4. Раздели задачу на этапы.
5. Начинай с минимально опасных изменений.
6. После каждого смыслового этапа проверяй, что ничего не сломано.
7. Если видишь системную проблему, не переписывай всё автоматически.
8. В конце дай понятный отчёт.

## Абсолютные правила
- Не делай огромные правки без необходимости
- Не ломай обратную совместимость без причины
- Не делай скрытый рефакторинг
- Не теряй фокус
