# Testing and Validation Skill

## Назначение
Этот навык нужен для того, чтобы не просто менять код, а подтверждать, что изменения корректны.

## Главный принцип
Любое изменение логики должно быть:
- проверяемым
- воспроизводимым
- объяснимым
- защищённым от регрессии

## Когда тесты обязательны
Если меняется хотя бы что-то из этого:
- формула
- коэффициент
- конвертация единиц
- запас
- правило округления
- упаковка
- формат итогового значения
- название или смысл поля, влияющего на расчёт
- логика выбора дефолтов
- поведение shared-core
- синхронизация между web и mobile
- исправление расчётного бага

## Минимальный набор тестов
1. Happy path
2. Zero or empty handling
3. Small values
4. Decimal values
5. Large values
6. Packaging rounding
7. Waste factor application
8. Regression test

## Что проверять особенно внимательно
- двойное округление
- двойной запас
- потерю единиц измерения
- разные результаты в web и mobile
- ломание соседних сценариев
